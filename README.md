# Internship Manager Tracker

A robust, type-safe web application for tracking internship applications, companies, and job positions. This project leverages the **Go + HTMX + Templ** stack to deliver a modern, interactive SPA-like experience without the complexity of a heavy frontend framework.

## 🚀 Key Features
- **User Authentication**: Secure registration and login with bcrypt password hashing and session management.
- **Data Isolation**: Multi-user support where applications, positions, and dashboard metrics are isolated per user.
- **Interactive Dashboard**: Real-time metrics and application pipeline visualization powered by HTMX.
- **Entity Management**: Complete CRUD for Companies and Job Positions.
- **Application Lifecycle**: Track application status (Applied, Interviewing, etc.) and schedule multiple interview stages.
- **Type-Safe Stack**: End-to-end type safety from the PostgreSQL schema (via `sqlc`) to the HTML templates (via `templ`).

## 🛠 Tech Stack
- **Language**: [Go 1.23+](https://go.dev/)
- **Router**: [Chi](https://github.com/go-chi/chi)
- **Templates**: [Templ](https://templ.guide/)
- **Frontend Interaction**: [HTMX](https://htmx.org/)
- **Database**: PostgreSQL 16
- **Database Tooling**: [sqlc](https://sqlc.dev/) (Type-safe SQL generation)
- **Database Driver**: [pgx/v5](https://github.com/jackc/pgx)
- **Styling**: Vanilla CSS & TailwindCSS (via CDN)

## 📦 Getting Started

### 1. Prerequisites
- Go 1.23+
- Docker & Docker Compose
- `templ` and `sqlc` CLIs (optional if using `go run` for tools)

### 2. Environment Setup
Clone the repository and start the database:
```bash
docker-compose up -d
```

Initialize the database schema:
```bash
# Using the Makefile
make db-init

# Or manually
psql "postgres://postgres:postgres@localhost:5432/internship_manager?sslmode=disable" -f sql/schema.sql
```

### 3. Build & Run
Generate code and start the server:
```bash
# Generate templ and sqlc code
make generate

# Run the server
make run
```
The application will be available at **http://localhost:8080**.

## 🏗 Project Structure
- `cmd/server/`: Main entry point and server initialization.
- `components/`: Type-safe UI components written in `.templ`.
- `internal/auth/`: Authentication context and session helpers.
- `internal/db/`: Database models and query logic generated by `sqlc`.
- `internal/handlers/`: Domain-specific HTTP handlers (Auth, Dashboard, Company, etc.).
- `sql/`: Raw SQL schema and query definitions.
- `static/`: Static assets including CSS.

## 📦 Codebase Documentation
This project follows the **"Cube Documentation Style"**. Every package, import block, struct, and function is documented with boxed comments that explicitly state the **PURPOSE** and **INFO** of the code block. This ensures high maintainability and immediate clarity for new contributors.

Example:
```go
/*
+--------------------------------------------------------------------------------+
| HANDLER: GET COUNTS                                                            |
| PURPOSE: Fetches high-level metrics for the dashboard header.                  |
| INFO: Returns an HTMX fragment containing total, applied, and interview counts.|
+--------------------------------------------------------------------------------+
*/
```
